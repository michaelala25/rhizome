# rhizome/agent/subagents/

Subagent base classes and specialized subagents.

## Architecture

Subagents are lightweight agent instances that run in isolated context windows, separate from the root agent's conversation history. They support stateful (multi-turn via `conversation_id`) and stateless modes. The preferred pattern is specialized subclasses with domain-specific invocation tools, rather than the generic `spawn_subagent`/`invoke_subagent` tools.

## Modules

- **base.py** — `Subagent` dataclass: a lightweight agent with its own isolated conversation history. Provides `preinvoke_hook`/`postinvoke_hook` for subclass customization. `ainvoke` accepts an optional `extra_state` dict for subagents with custom state schemas (e.g. the commit subagent passes `commit_proposal`), and returns `(conversation_id, ai_message, result_state)`. `StructuredSubagent` extends `Subagent` with JSON response parsing into a `response_schema` type (dataclass or Pydantic model), storing the result in `structured_response`.
- **commit.py** — Commit workflow for extracting knowledge entries from selected conversation messages. Defines `CommitSubagentState` (extends `AgentState` with `commit_proposal`), `CommitEntryEdit` and `KnowledgeEntryProposalSchema` Pydantic schemas, and `_build_commit_diff` helper. `build_commit_subagent()` creates a `Subagent` with DATABASE tools and internal proposal tools (`stage_entries`, `edit_entries`) that operate on `CommitSubagentState`. `build_commit_subagent_tools()` returns six tools for the root agent: `commit_show_selected_messages`, `commit_invoke_subagent` (delegates to the subagent, passing current proposal + diff via `extra_state`), `commit_proposal_create` (direct path for small selections), `commit_proposal_present` (shows proposal via interrupt, writes diff to `proposal_diff` on Edit), `commit_proposal_edit` (targeted edits by stable ID), `commit_proposal_accept` (writes to DB). All tools use `ToolRuntime` to read/write `commit_proposal_state` in `RhizomeAgentState` via `Command(update={...})`.
- **flashcard_validator.py** — Flashcard quality validation and review scoring via toolless subagents. `build_answerer_subagent()` creates a `StructuredSubagent` (Haiku, temp=0) that answers each flashcard question with no additional context. `build_comparator_subagent()` creates a `StructuredSubagent` (Sonnet, temp=0) that compares expected answers against the answerer's responses and evaluates clarity/unambiguity. These are passed to `build_flashcard_proposal_tools()` and invoked inline when `flashcard_proposal_create` is called with `validate=True`. `build_scorer_subagent()` creates a `StructuredSubagent` (Haiku, temp=0) that scores user answers against expected answers on a 1-4 scale (1=again, 2=hard, 3=good, 4=easy); passed to `build_review_tools()` and invoked inline by `review_present_flashcards` for cards the user marked "auto".
